# Product Requirements Document

## 1. Overview
This product is a dedicated landing page and integrated Content Management System (CMS) specifically tailored for Ladang Lima, a gluten-free food company. It targets Ladang Lima's marketing team, solving the problem of presenting dynamic product information, engaging recipes, and brand stories without developer intervention. The product empowers Ladang Lima to manage product showcases, recipe content, and promotional materials dynamically, thereby enhancing online presence and reducing operational overhead.

## 2. Tech Stack
The frontend will be built with Next.js for server-side rendering and optimal performance, leveraging TanStack Query for efficient data fetching and caching. The backend will be powered by Laravel, providing robust API endpoints for content, product, and recipe management, and serving the CMS interface. MySQL will serve as the primary database for storing all content and user data. The frontend will be hosted on Vercel for seamless deployment and performance, while the Laravel backend will reside on cPanel. Styling will likely utilize Tailwind CSS for rapid UI development, allowing for flexible theming to match Ladang Lima's brand.

## 3. Features
CMS Module: - User Authentication & Authorization (MVP) - Dashboard for content overview - Homepage Content Management: Edit hero banners (e.g., new product launches, promotions), Product Showcase sections (e.g., best sellers, new arrivals), Recipe Highlights, Brand Story, Testimonials/Customer Stories (MVP) - Button Management: Add, edit, delete call-to-action buttons with custom links (MVP) - Product Management: Create, edit, publish, unpublish, and delete products (name, description, ingredients, nutritional info, images) (MVP) - Recipe Management: Create, edit, publish, unpublish, and delete recipes (title, ingredients list, instructions, images, prep/cook time) (MVP) - Article Post Management: Create, edit, publish, unpublish, and delete general articles (e.g., health benefits, company news) (MVP) - Media Library: Upload and manage images/videos for products, recipes, articles, and pages (MVP) - SEO Settings: Meta title, description, and slug management for pages, products, recipes, and articles - Content Versioning (future) - Role-Based Access Control (future) Public Landing Page Module: - Dynamic Homepage Display: Renders content like hero banners, product categories, recipe cards, brand story, testimonials from CMS (MVP) - Dynamic Product Listing Page: Displays all Ladang Lima products, with filtering by category (MVP) - Individual Product Detail Pages: Renders full product information, ingredients, nutritional facts, images, and purchase links (MVP) - Dynamic Recipe Listing Page: Displays all published recipes (MVP) - Individual Recipe Pages: Renders full recipe content with images and step-by-step instructions (MVP) - Dynamic Article Listing Page: Displays all published articles (MVP) - Individual Article Pages: Renders full article content (MVP) - Responsive Design for all devices (MVP) - Contact Form integration (future) - Analytics Integration (future)

## 4. Data Model
Main Tables: - Users: id (PK), name, email, password, role, created_at, updated_at - Pages: id (PK), title, slug (unique), content_json (JSON for dynamic sections), status (draft/published), created_at, updated_at - PageSections: id (PK), page_id (FK to Pages), section_key (e.g., 'hero', 'products_showcase', 'recipes_highlight', 'brand_story'), content_html (or markdown), order, created_at, updated_at - Buttons: id (PK), page_section_id (FK to PageSections, nullable), text, url, css_class, created_at, updated_at - Products: id (PK), name, slug (unique), description, ingredients, nutritional_info, image_url, status (draft/published), category_id (FK to Categories), created_at, updated_at - Recipes: id (PK), title, slug (unique), description, ingredients_list (JSON/text), instructions (JSON/text), image_url, prep_time, cook_time, serving_size, status (draft/published), created_at, updated_at - ArticlePosts: id (PK), title, slug (unique), content, author_id (FK to Users), published_at, status (draft/published), image_url, meta_description, created_at, updated_at - Categories: id (PK), name, slug, type (e.g., 'product', 'article'), created_at, updated_at - ProductCategories (Pivot Table): product_id (FK to Products), category_id (FK to Categories where type='product') - ArticlePostCategories (Pivot Table): article_post_id (FK to ArticlePosts), category_id (FK to Categories where type='article')

## 5. Phases
Phase 1: Planning & Design (2 weeks) - Detailed requirements analysis for Ladang Lima's content needs (products, recipes, articles), user stories, and feature prioritization. - UI/UX design for both public landing page and CMS admin panel (wireframes, mockups) ensuring brand consistency with Ladang Lima's visual identity. - Database schema design and API contract definition for product, recipe, and content management. - Infrastructure planning and initial setup (repo, basic deployments). Phase 2: Core Development (6 weeks) - Backend setup with Laravel, including API endpoints for User, Pages, Products, Recipes, ArticlePosts, and CMS functionality. - Frontend development with Next.js for the dynamic public landing page showcasing products, recipes, and articles. - Implementation of User Authentication and Authorization. - Development of the core CMS modules for Homepage content, Button, Product, Recipe, and Article post management (CRUD). - Integration of frontend with backend APIs. Phase 3: Testing & Deployment (2 weeks) - Comprehensive unit, integration, and end-to-end testing across all modules. - User Acceptance Testing (UAT) with Ladang Lima's internal stakeholders to ensure content management ease and public site accuracy. - Performance testing, security audits, and bug fixing. - Deployment to production environments (Vercel for frontend, cPanel for backend). Phase 4: Post-Launch & Iteration (Ongoing) - Continuous monitoring, performance optimization, and bug resolution. - Implementation of advanced features like Media Library, Content Versioning, and enhanced SEO. - Gathering feedback from Ladang Lima and planning for future iterations, such as e-commerce integration or advanced analytics. - Scaling infrastructure based on traffic and content growth.
